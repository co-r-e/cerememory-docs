This guide walks through the core Cerememory workflow: start the server, store a memory, recall it, and remove it.

## Start the Server

<Tabs items="CLI,Docker">

<div>

```bash
cerememory serve --port 8420
```

Example startup output:

```text
Cerememory v0.2.2
HTTP  listening on 127.0.0.1:8420
```

</div>

<div>

```bash
docker run --rm -p 8420:8420 ghcr.io/co-r-e/cerememory:latest
```

</div>

</Tabs>

By default Cerememory starts with:

- HTTP bound to `127.0.0.1:8420`
- Background decay enabled
- Persistent `redb` stores for episodic, semantic, procedural, and emotional memory
- In-memory working memory
- `/health` and `/readiness` enabled
- `/metrics` disabled until `http.metrics_enabled = true`
- Background dream processing enabled by default (`dream.background_interval_secs = 86400`)

<Callout type="info">
For agent workflows, run `cerememory mcp` to expose the MCP tool surface over stdio instead of starting the HTTP server. This works with any MCP-compatible client — Claude Code, Codex CLI, Cursor, Cline, Windsurf, Zed, Continue, and others. See [MCP & Agent Setup](/docs/getting-started/agent-memory) for per-client configuration.
</Callout>

## Store a Memory

<Tabs items="Python,TypeScript,curl,CLI">

<div>

```python
from cerememory import Client

client = Client("http://localhost:8420")

record_id = client.store(
    "The user prefers dark mode and uses vim keybindings.",
    store="semantic",
    metadata={"source": "onboarding"},
)

print(f"Stored: {record_id}")
```

</div>

<div>

```typescript
import { CerememoryClient } from "@cerememory/sdk";

const client = new CerememoryClient("http://localhost:8420");

const recordId = await client.store(
  "The user prefers dark mode and uses vim keybindings.",
  {
    store: "semantic",
    metadata: { source: "onboarding" },
  }
);

console.log(`Stored: ${recordId}`);
```

</div>

<div>

```bash
curl -X POST http://localhost:8420/v1/encode \
  -H "Content-Type: application/json" \
  -d '{
    "content": {
      "blocks": [{
        "modality": "text",
        "format": "text/plain",
        "data": [84,104,101,32,117,115,101,114,32,112,114,101,102,101,114,115,32,100,97,114,107,32,109,111,100,101,32,97,110,100,32,117,115,101,115,32,118,105,109,32,107,101,121,98,105,110,100,105,110,103,115,46],
        "embedding": null
      }],
      "summary": null
    },
    "store": "semantic",
    "metadata": {
      "source": "onboarding"
    }
  }'
```

</div>

<div>

```bash
cerememory store "The user prefers dark mode and uses vim keybindings." \
  --store semantic
```

</div>

</Tabs>

## Recall Memories

<Tabs items="Python,TypeScript,curl,CLI">

<div>

```python
memories = client.recall(
    "What are the user's preferences?",
    limit=5,
    mode="human",
)

for memory in memories:
    print(f"[{memory.record.store}] score={memory.relevance_score:.4f}")
```

</div>

<div>

```typescript
const memories = await client.recall("What are the user's preferences?", {
  limit: 5,
  recall_mode: "human",
});

for (const memory of memories) {
  console.log(`[${memory.record.store}] score=${memory.relevance_score.toFixed(4)}`);
}
```

</div>

<div>

```bash
curl -X POST http://localhost:8420/v1/recall/query \
  -H "Content-Type: application/json" \
  -d '{
    "cue": { "text": "What are the user preferences?" },
    "limit": 5,
    "recall_mode": "human"
  }'
```

</div>

<div>

```bash
cerememory recall "What are the user's preferences?" --limit 5 --mode human
```

</div>

</Tabs>

<Callout type="info">
In **human** mode, Cerememory returns `rendered_content` with fidelity and noise applied. Use **perfect** mode when you need exact stored content.
</Callout>

## Preserve Raw Journal Entries

Raw journal records keep the verbatim source material that dream processing can later summarize into curated episodic and semantic memory.

<Tabs items="Python,TypeScript,CLI">

<div>

```python
raw_id = client.store_raw(
    "The user said they prefer Neovim and keep tmux open all day.",
    session_id="sess_001",
    source="conversation",
    speaker="user",
)

dream = client.dream_tick(session_id="sess_001")
print(raw_id, dream.episodic_summaries_created)
```

</div>

<div>

```typescript
const rawId = await client.storeRaw(
  "The user said they prefer Neovim and keep tmux open all day.",
  {
    sessionId: "sess_001",
    source: "conversation",
    speaker: "user",
  }
);

const dream = await client.dreamTick({ session_id: "sess_001" });
console.log(rawId, dream.episodic_summaries_created);
```

</div>

<div>

```bash
cerememory store-raw \
  "The user said they prefer Neovim and keep tmux open all day." \
  --session-id sess_001

cerememory dream-tick --session-id sess_001
```

</div>

</Tabs>

<Callout type="info">
For agent-first workflows, `cerememory mcp` exposes the same memory system over stdio so Claude Code, OpenAI Codex CLI, Cursor, and any other MCP-compatible client can use `store`, `store_raw`, `recall`, `recall_raw`, `dream_tick`, `consolidate`, `export`, and related tools directly.
</Callout>

## Forget a Memory

<Tabs items="Python,TypeScript,CLI">

<div>

```python
client.forget(record_id, confirm=True)
```

</div>

<div>

```typescript
await client.forget(recordId, { confirm: true });
```

</div>

<div>

```bash
cerememory forget 01916e3a-1234-7000-8000-000000000001 --confirm
```

</div>

</Tabs>

<Callout type="warning">
Forget is irreversible. Use `--cascade` or `cascade=True` only when you intentionally want associated records deleted as well.
</Callout>

## Check System State

```bash
curl http://localhost:8420/readiness
curl http://localhost:8420/v1/introspect/stats
```

`/readiness` is unauthenticated and intended for probes. `/v1/introspect/stats` is the full CMP stats endpoint.
