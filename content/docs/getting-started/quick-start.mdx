This guide walks through the core Cerememory workflow: start one long-lived server for your data directory, point every MCP client (or any HTTP/gRPC client) at it, and store, recall, and forget a memory.

## Start the Shared Server

Cerememory's only supported operating mode is **one long-lived server process per data directory**. The server owns the embedded `redb` and Tantivy stores exclusively. Every MCP client, CLI invocation, and HTTP caller then talks to that shared server.

```bash
target/release/cerememory serve --data-dir ~/.cerememory/data
```

Example startup output:

```text
Cerememory v0.2.3
HTTP  listening on 127.0.0.1:8420
```

Default behavior:

- HTTP bound to `127.0.0.1:8420`
- Persistent `redb` stores for episodic, semantic, procedural, emotional, and raw journal memory
- In-memory working memory (never persisted)
- Background decay tick (`decay.background_interval_secs = 3600`)
- Background dream processing (`dream.background_interval_secs = 86400`)
- `/health` and `/readiness` enabled
- `/metrics` disabled until `http.metrics_enabled = true`

<Callout type="warning">
Only one process may open a given `data_dir` at a time. Starting a second `cerememory serve` or an embedded-mode client on the same directory will fail because `redb` holds an exclusive lock. If you need multi-client access, use the shared MCP setup below instead of running multiple servers.
</Callout>

## Connect an MCP Client

`cerememory mcp` is a thin stdio proxy that speaks MCP to the client and CMP/HTTP to the shared server. `--server-url` is **required**.

```bash
target/release/cerememory mcp --server-url http://127.0.0.1:8420
```

Point every MCP-compatible client at that same server. The same binary works with Claude Code, OpenAI Codex CLI, Cursor, Cline, Windsurf, Zed, Continue, and any other client that speaks MCP.

<Callout type="info">
If the shared server requires auth, set `CEREMEMORY_SERVER_API_KEY` in the MCP client's environment instead of passing the token on the command line. For long-running upstream operations, use `--server-timeout-secs`; omitting it disables per-request timeouts.
</Callout>

See [MCP & Agent Setup](/docs/getting-started/agent-memory) for per-client configuration snippets.

## Store a Memory

<Tabs items="MCP,CLI,curl">

<div>

Once a client is connected, call the `store` tool:

```text
store(
  content="The user prefers dark mode and uses vim keybindings.",
  store="semantic",
  metadata={"source": "onboarding"}
)
```

</div>

<div>

```bash
target/release/cerememory store \
  "The user prefers dark mode and uses vim keybindings." \
  --store semantic
```

</div>

<div>

```bash
curl -X POST http://127.0.0.1:8420/v1/encode \
  -H "Content-Type: application/json" \
  -d '{
    "content": {
      "blocks": [{
        "modality": "text",
        "format": "text/plain",
        "data": [84,104,101,32,117,115,101,114,32,112,114,101,102,101,114,115,32,100,97,114,107,32,109,111,100,101,32,97,110,100,32,117,115,101,115,32,118,105,109,32,107,101,121,98,105,110,100,105,110,103,115,46],
        "embedding": null
      }],
      "summary": null
    },
    "store": "semantic",
    "metadata": {
      "source": "onboarding"
    }
  }'
```

</div>

</Tabs>

## Recall Memories

<Tabs items="MCP,CLI,curl">

<div>

```text
recall(query="What are the user's preferences?", limit=5, mode="human")
```

</div>

<div>

```bash
target/release/cerememory recall "What are the user's preferences?" --limit 5 --mode human
```

</div>

<div>

```bash
curl -X POST http://127.0.0.1:8420/v1/recall/query \
  -H "Content-Type: application/json" \
  -d '{
    "cue": { "text": "What are the user preferences?" },
    "limit": 5,
    "recall_mode": "human"
  }'
```

</div>

</Tabs>

<Callout type="info">
In **human** mode, Cerememory returns `rendered_content` with fidelity and noise applied. Use **perfect** mode when you need exact stored content.
</Callout>

## Preserve Raw Journal Entries

Raw journal records keep the verbatim source material that dream processing can later summarize into curated episodic and semantic memory.

<Tabs items="MCP,CLI">

<div>

```text
store_raw(
  text="The user said they prefer Neovim and keep tmux open all day.",
  session_id="sess_001",
  source="conversation",
  speaker="user"
)

dream_tick(session_id="sess_001")
```

</div>

<div>

```bash
target/release/cerememory store-raw \
  "The user said they prefer Neovim and keep tmux open all day." \
  --session-id sess_001

target/release/cerememory dream-tick --session-id sess_001
```

</div>

</Tabs>

## Forget a Memory

<Tabs items="MCP,CLI">

<div>

```text
forget(record_ids=["01916e3a-1234-7000-8000-000000000001"], confirm=true)
```

</div>

<div>

```bash
target/release/cerememory forget 01916e3a-1234-7000-8000-000000000001 --confirm
```

</div>

</Tabs>

<Callout type="warning">
Forget is irreversible. Use `--cascade` (CLI) or `cascade=true` (MCP/HTTP) only when you intentionally want associated records deleted as well.
</Callout>

## Check System State

```bash
curl http://127.0.0.1:8420/readiness
curl http://127.0.0.1:8420/v1/introspect/stats
```

`/readiness` is unauthenticated and intended for probes. `/v1/introspect/stats` is the full CMP stats endpoint.
