## Requirements

- **Rust 1.88+** for building the CLI from source
- **Python 3.9+** for the Python SDK and native bindings
- **Node.js 18+** for the TypeScript SDK and Node native bindings
- No external database required -- Cerememory uses embedded `redb` for persistent storage

## Installation Methods

<Tabs items="cargo,Docker,pip,npm">

<div>

Install the CLI from crates.io:

```bash
cargo install cerememory-cli
```

Build with only the adapters you need:

```bash
cargo install cerememory-cli --no-default-features --features llm-claude
```

</div>

<div>

Pull the published container image:

```bash
docker pull ghcr.io/co-r-e/cerememory:latest
```

Run it with a persistent volume:

```bash
docker run --rm \
  -p 8420:8420 \
  -v cerememory-data:/data \
  ghcr.io/co-r-e/cerememory:latest
```

</div>

<div>

Install the HTTP client SDK:

```bash
pip install cerememory
```

Build the in-process native binding from `bindings/python-native` with `maturin` when you need embedded, no-HTTP access.

</div>

<div>

Install the HTTP client SDK:

```bash
npm install @cerememory/sdk
```

Build the in-process native binding from `bindings/typescript-native` when you need embedded, no-HTTP access.

</div>

</Tabs>

## Build from Source

```bash
git clone https://github.com/co-r-e/cerememory.git
cd cerememory
cargo build -p cerememory-cli --release
```

The binary will be available at `target/release/cerememory`.

## Feature Flags

The CLI enables all LLM adapters by default. You can trim binary size by disabling unused providers.

| Feature | Description | Default |
|---------|-------------|---------|
| `llm-openai` | OpenAI / GPT adapter | Enabled |
| `llm-claude` | Anthropic / Claude adapter | Enabled |
| `llm-gemini` | Google / Gemini adapter | Enabled |

## Verify Installation

```bash
cerememory --version
```

You should see output like:

```text
cerememory 0.2.0
```

<Callout type="info">
The CLI and server are currently `v0.2.0`. The language SDK packages may publish on their own cadence, so prefer the package manager output for exact SDK package versions.
</Callout>
