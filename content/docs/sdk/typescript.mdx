## Installation

```bash
npm install @cerememory/sdk
```

The SDK uses native `fetch`, so Node.js `18+` or a browser/runtime with `fetch` is required.

## Client Setup

```typescript
import { CerememoryClient } from "@cerememory/sdk";

const client = new CerememoryClient("http://localhost:8420", {
  apiKey: "sk-cerememory-prod-abc123",
  timeoutMs: 30_000,
  maxRetries: 0,
  retryMutatingRequests: false,
});
```

<Callout type="info">
Retries are conservative by default in `0.2.0`: `maxRetries` defaults to `0`, and mutating requests are retried only when `retryMutatingRequests` is explicitly enabled.
</Callout>

## High-Level Methods

<Callout type="info">
The high-level helpers are text-first conveniences. For multimodal content, manual associations, or full CMP request bodies, use methods such as `encodeStore()` or `recallQuery()`.
</Callout>

### `store(text, options)`

Store a text memory and receive its UUID.

```typescript
const recordId = await client.store(
  "The user prefers TypeScript for new projects.",
  {
    store: "semantic",
    metadata: { source: "onboarding" },
    emotion: { trust: 1, intensity: 0.6, valence: 0.7 },
  }
);
```

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `store` | `StoreType` | `episodic` | Target store |
| `emotion` | `Partial<EmotionVector>` | None | Emotion values merged into a full vector |
| `context` | `EncodeContext` | None | Session, spatial, temporal, or source context |
| `metadata` | `Record<string, unknown>` | None | Stored with the record |

### `recall(query, options)`

Recall memories matching a text cue.

```typescript
const memories = await client.recall(
  "What tooling does the user prefer?",
  {
    limit: 5,
    stores: ["semantic", "episodic"],
    recall_mode: "human",
    reconsolidate: true,
    activation_depth: 2,
  }
);
```

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `limit` | `number` | `10` | Maximum number of results |
| `stores` | `StoreType[]` | All stores | Restrict the search |
| `min_fidelity` | `number` | None | Minimum fidelity threshold |
| `recall_mode` | `RecallMode` | `human` | `human` or `perfect` |
| `include_decayed` | `boolean` | `false` | Include records below the normal prune threshold |
| `reconsolidate` | `boolean` | `true` | Strengthen recalled memories |
| `activation_depth` | `number` | `2` | Spreading activation depth |

### `forget(recordId, options)`

Delete a memory record.

```typescript
await client.forget(recordId, {
  confirm: true,
  cascade: false,
});
```

### `stats()` and `health()`

```typescript
const stats = await client.stats();
const health = await client.health();
```

`stats()` returns the full `StatsResponse`, including `background_decay_enabled` and optional `evolution_metrics`.

## Full CMP Methods

The client also exposes protocol-level methods for the full REST surface:

| Method | Purpose |
|--------|---------|
| `encodeStore()` | `POST /v1/encode` |
| `encodeBatch()` | `POST /v1/encode/batch` |
| `encodeUpdate(recordId, request)` | `PATCH /v1/encode/:record_id` |
| `recallQuery()` | `POST /v1/recall/query` |
| `recallAssociate(recordId, request)` | `POST /v1/recall/associate/:record_id` |
| `recallTimeline()` | `POST /v1/recall/timeline` |
| `recallGraph()` | `POST /v1/recall/graph` |
| `lifecycleConsolidate()` | `POST /v1/lifecycle/consolidate` |
| `lifecycleDecayTick()` | `POST /v1/lifecycle/decay-tick` |
| `lifecycleSetMode()` | `PUT /v1/lifecycle/mode` |
| `lifecycleForget()` | `DELETE /v1/lifecycle/forget` |
| `introspectStats()` | `GET /v1/introspect/stats` |
| `introspectRecord(recordId)` | `GET /v1/introspect/record/:record_id` |
| `introspectDecayForecast()` | `POST /v1/introspect/decay-forecast` |
| `introspectEvolution()` | `GET /v1/introspect/evolution` |

Example:

```typescript
const graph = await client.recallGraph({
  center_id: recordId,
  depth: 2,
  limit_nodes: 25,
});
```

## Vercel AI SDK Integration

The package exports `CerememoryProvider` for Vercel AI SDK tool definitions:

```typescript
import { CerememoryClient, CerememoryProvider } from "@cerememory/sdk";

const client = new CerememoryClient("http://localhost:8420");
const provider = new CerememoryProvider(client);

const tools = provider.tools();
const augmentedPrompt = await provider.augmentPrompt(
  "You are a helpful assistant.",
  "user preferences",
  5
);
```

## Error Handling

```typescript
import {
  CerememoryError,
  RecordNotFoundError,
  RateLimitedError,
  UnauthorizedError,
} from "@cerememory/sdk";

try {
  await client.introspectRecord("nonexistent-id");
} catch (error) {
  if (error instanceof RecordNotFoundError) {
    console.error("Record not found");
  } else if (error instanceof RateLimitedError) {
    console.error(`Retry after ${error.retryAfter}s`);
  } else if (error instanceof UnauthorizedError) {
    console.error("Missing or invalid API key");
  } else if (error instanceof CerememoryError) {
    console.error(`${error.code}: ${error.message}`);
  }
}
```

Errors surface request IDs when the server returns them, which helps correlate SDK failures with server logs.
