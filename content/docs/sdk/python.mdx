## Installation

```bash
pip install cerememory
```

For the native in-process binding, see [Native Bindings](/docs/sdk/native-bindings).

## Sync Client

```python
from cerememory import Client

client = Client(
    "http://localhost:8420",
    api_key="sk-cerememory-prod-abc123",
    timeout=30.0,
    max_retries=0,
    retry_mutating_requests=False,
)
```

<Callout type="info">
Retries are conservative by default in `0.2.0`: `max_retries` defaults to `0`, and mutating requests are retried only when `retry_mutating_requests=True`.
</Callout>

## High-Level Methods

<Callout type="info">
The high-level helpers are text-first conveniences. For multimodal blocks, manual associations, full encode context, or custom `MemoryContent`, use the protocol-level methods such as `encode_store()`.
</Callout>

### `store(text, ...)`

Store a text memory and receive its UUID.

```python
record_id = client.store(
    "The user's preferred editor is Neovim.",
    store="semantic",
    metadata={"source": "onboarding"},
)
print(record_id)
```

Supported keyword arguments:

| Argument | Type | Default | Description |
|----------|------|---------|-------------|
| `store` | `str | StoreType` | `StoreType.EPISODIC` | Target store |
| `emotion` | `EmotionVector | None` | None | Optional emotion vector |
| `metadata` | `dict | None` | None | Stored with the record |

### `recall(query, ...)`

Recall memories matching a text cue.

```python
memories = client.recall(
    "What editor does the user prefer?",
    limit=5,
    stores=["semantic", "episodic"],
    mode="human",
    reconsolidate=True,
    activation_depth=2,
)

for memory in memories:
    print(memory.record.store, memory.relevance_score)
```

| Argument | Type | Default | Description |
|----------|------|---------|-------------|
| `limit` | `int` | `10` | Maximum number of results |
| `stores` | `list[str | StoreType] | None` | None | Restrict the search |
| `mode` | `str | RecallMode` | `RecallMode.HUMAN` | `human` or `perfect` |
| `min_fidelity` | `float | None` | None | Minimum fidelity threshold |
| `reconsolidate` | `bool` | `True` | Strengthen recalled memories |
| `activation_depth` | `int` | `2` | Spreading activation depth |

### `forget(*record_ids, ...)`

```python
deleted = client.forget(record_id, confirm=True, cascade=False)
print(deleted)
```

### `stats()`, `get_record()`, `health()`, `readiness()`

```python
stats = client.stats()
record = client.get_record(record_id)
health = client.health()
readiness = client.readiness()
```

`stats()` returns `background_decay_enabled` and optional `evolution_metrics` in addition to the usual record counters.

## Full CMP Methods

The sync client also exposes protocol-level methods:

| Method | Purpose |
|--------|---------|
| `encode_store()` | `POST /v1/encode` |
| `encode_batch()` | `POST /v1/encode/batch` |
| `encode_update(record_id, request)` | `PATCH /v1/encode/:record_id` |
| `recall_query()` | `POST /v1/recall/query` |
| `recall_associate(record_id, request)` | `POST /v1/recall/associate/:record_id` |
| `recall_timeline()` | `POST /v1/recall/timeline` |
| `recall_graph()` | `POST /v1/recall/graph` |
| `lifecycle_consolidate()` | `POST /v1/lifecycle/consolidate` |
| `lifecycle_decay_tick()` | `POST /v1/lifecycle/decay-tick` |
| `lifecycle_set_mode()` | `PUT /v1/lifecycle/mode` |
| `lifecycle_forget()` | `DELETE /v1/lifecycle/forget` |
| `introspect_stats()` | `GET /v1/introspect/stats` |
| `introspect_record(record_id)` | `GET /v1/introspect/record/:record_id` |
| `introspect_decay_forecast()` | `POST /v1/introspect/decay-forecast` |
| `introspect_evolution()` | `GET /v1/introspect/evolution` |

Example:

```python
metrics = client.introspect_evolution()
print(metrics.parameter_adjustments)
```

## Async Client

The async client mirrors the sync API:

```python
from cerememory import AsyncClient

async with AsyncClient("http://localhost:8420") as client:
    record_id = await client.store("Async memory", store="episodic")
    memories = await client.recall("async memory")
    await client.forget(record_id, confirm=True)
```

Use `AsyncClient` for asyncio-based services, agents, or web backends.

## Integrations

The Python package also includes integration modules for:

- `cerememory.integrations.langchain`
- `cerememory.integrations.llamaindex`

Use these when you want Cerememory-backed memory or vector-store behavior inside LangChain or LlamaIndex applications.

## Error Handling

```python
from cerememory import (
    CerememoryError,
    RecordNotFoundError,
    RateLimitedError,
    UnauthorizedError,
)

try:
    client.get_record("00000000-0000-0000-0000-000000000000")
except RecordNotFoundError:
    print("Record not found")
except RateLimitedError as exc:
    print(exc.retry_after)
except UnauthorizedError:
    print("Missing or invalid API key")
except CerememoryError as exc:
    print(exc.code, exc.message)
```
